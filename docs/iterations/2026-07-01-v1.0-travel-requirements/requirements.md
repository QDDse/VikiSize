# v1.0 旅行功能聚合需求

## 1. 目标

让用户能够先浏览一份完整、美观、可信的旅行攻略，再把它变成自己旅行空间中可调整、可协作、可执行的计划。

v1.0 必须完成一条端到端链路：

`模板列表 -> 模板详情 -> H5 完整攻略 -> 创建实例 -> 原生编辑 -> 今日执行`

## 2. 用户与权限

- 管理员：创建空间、使用模板、管理成员、编辑全部旅行内容、归档旅行。
- 成员：编辑旅行实例、节点、模块、任务、图片、提醒、评论和意见。
- 访客：只读浏览被邀请空间中的旅行内容。
- 访客默认看不到确认号、证件信息和标记为敏感的内部预算。

## 3. 旅行模板列表

每张模板卡必须展示：

- 封面图。
- 标题。
- 简介 `desc`。
- 天数、主要目的地、适合人群、节奏和出发月份标签。
- 模板版本与更新时间。
- `浏览攻略`、`使用此计划`；已有实例时提供 `继续编辑`。

东京模板简介：

> 给两个人安排的东京进出方案，覆盖市区经典、台场夜景、镰仓海边、箱根温泉与迪士尼海洋；户外尽量安排在上午和傍晚。

## 4. 原生模板详情

- 显示封面、简介、8 天主题摘要、预算参考、地点数、预约项数量和免责声明。
- 主操作为 `使用此计划`。
- 次操作为 `浏览完整攻略`。
- H5 加载失败不能阻塞用户创建实例。

## 5. H5 完整攻略

- 使用 HTTPS 托管，并通过小程序 `web-view` 打开。
- 保留现有 HTML 的封面、待办、行前须知、航班酒店、地图、时间线、餐饮、替代方案、贴士和来源。
- H5 只展示公共模板，不读取或编辑空间实例。
- H5 与模板详情展示相同 `templateId` 和 `templateVersion`。
- 预览地址不可用时提供原生降级说明和重试。

## 6. 单一模板数据源

- 从 HTML 内联 `trip` 提取版本化结构数据。
- H5 和小程序模板都由同一份数据生成。
- 每个日期、节点、餐饮、住宿和预订项拥有稳定 ID。
- 模板至少包含当前 HTML 已有信息：行前须知、航班、住宿、提醒、每日节点、坐标、图片、评分、营业时间、门票、交通、餐饮、替代方案、贴士和免责声明。
- 模板升级只影响新实例，不自动覆盖已有实例。

## 7. 创建旅行实例

- 用户选择新建旅行空间或使用已有旅行空间。
- 通过 `templateId + templateVersion + spaceId` 创建实例。
- 创建操作必须幂等，同一空间重复点击不能产生重复实例。
- 实例保存来源模板、来源版本、导入时间和初始快照。
- 修改实例不影响模板、公共 H5 或其他空间实例。

## 8. 原生旅行编辑器

### 日期与行程节点

- 新增、编辑、删除、复制和排序日期。
- 新增、编辑、删除、复制和排序节点。
- 节点字段：类型、标题、开始时间、结束时间或停留时长、地点、地址、坐标、交通、预约、预算、图片、备注和敏感字段。

### 模块

- 景点或活动。
- 航班。
- 住宿。
- 餐饮。
- 交通。
- 自由备注。
- 备选方案。
- 任务、提醒和待确认项。

### 候选地点池

- 用户可以先保存地点，不立即安排日期。
- 候选地点可以排入某一天，也可以移回待安排状态。
- 排期不得产生无来源关系的重复地点数据。

## 9. 地图与交通

- 地图和时间线读取同一批节点 ID 和排序。
- 地图可以按天显示和隐藏点位。
- 点击节点时地图高亮对应点位，点击点位时定位对应节点。
- 相邻节点保存交通方式，并提供打开微信地图的入口。
- 自动计算交通时间、距离和路线优化不作为 v1.0 P0 阻塞项。

## 10. 派生任务与协作

- 行程节点可以派生订票、酒店、证件、打包、交通和待确认任务。
- 派生任务保留 `sourceType` 和 `sourceId`。
- 节点变更时只提示相关任务可能需要同步，不静默覆盖成员修改。
- 关键编辑记录活动。
- 评论、成员意见、图片和提醒沿用空间统一能力。

## 11. 今日执行视图

旅行期间今日页优先显示：

- 当前节点和下一站。
- 时间、地址、交通和打开地图。
- 相关票据图片和预约状态。
- 今日待确认、提醒和备选方案。
- 编辑入口默认收起。

## 12. 归档

- 旅行结束后可以归档整个旅行实例。
- 归档保留节点、任务、预算、评论、活动和图片。
- 归档旅行默认不出现在今日页，但可以查看和恢复。

## 13. P0、P1 和非目标

### P0

- 模板列表、详情和 `desc`。
- H5 只读预览。
- 单一模板数据源。
- 幂等实例创建。
- 原生日期、节点和模块编辑。
- 候选地点池。
- 时间线与地图同源。
- 派生任务、权限、图片、提醒和归档。

### P1

- 相邻地点自动计算时间和距离。
- 单日路线优化、变化预览和撤销。
- 弱网缓存关键行程和附件缩略图。
- 面向同行人的精简分享页。

### 非目标

- H5 直接编辑空间实例。
- H5 与小程序实时双向同步。
- AI 自动生成或重排行程。
- 邮件自动解析预订。
- 实时机酒价格、代订和支付。
- 复杂费用分摊。
- 模板自动覆盖已有实例。
- 菜谱功能实现；菜谱需求保留到后续生活模块版本。

## 14. 总体验收

- 用户从列表看到东京模板简介，并能打开完整 H5 攻略。
- H5 不可用时用户仍能创建实例。
- 用户使用模板后获得独立、可编辑的旅行实例。
- 管理员和成员可以增删改排节点，访客不能修改。
- 地图和时间线的节点数量、ID 与排序一致。
- 派生任务可以追溯到来源节点。
- 模板、H5 和实例初始内容使用相同模板版本。
- 修改实例不会改变模板、H5 和其他空间。
- 旅行归档后不再出现在今日页，恢复后重新可见。
