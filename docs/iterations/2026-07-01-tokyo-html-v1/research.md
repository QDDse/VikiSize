# 本轮调研：东京 HTML 接入小程序

> 状态：源调研材料。已聚合到 `docs/iterations/2026-07-01-v1.0-travel-requirements/`。

## 输入

- `关东东京8天旅行计划.html`
- 现有 `tokyoTravelTemplate.js`
- 现有旅行计划列表与按天页面
- 2026-07-01 旅行产品调研结论

## 发现

- HTML 已经是完整只读攻略，包含的字段显著多于小程序模板。
- 小程序已有模板实例架构，适合承接编辑，但节点编辑目前只更新确认备注。
- 直接把 HTML 变成编辑器会把权限、同步、附件和地图编辑放进 H5，复杂度过高。
- 只做原生页面会丢失现有 HTML 的视觉成果。
- 当前 GitHub Pages 根地址可访问，但正式小程序接入还需要业务域名和真机验证。

## 决策

采用“单一模板 JSON + H5 只读预览 + 小程序原生编辑器”。

完整结论已收录到 `docs/versions/v1.0/`。
